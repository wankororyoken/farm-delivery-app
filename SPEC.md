# 農場配達管理アプリ 仕様書

## 概要

農場で生産した野菜を複数の納品先に配達する際の納品データを記録・管理するウェブアプリ。
Google Apps Script (GAS) でホスティングし、バックエンドのデータストアとして Google スプレッドシートを使用する。
iPhone での現場入力を主な用途として設計されている。

---

## 技術スタック

| レイヤー | 技術 |
|---|---|
| ホスティング | Google Apps Script Web App (`doGet()`) |
| フロントエンド | Vanilla HTML / CSS / JavaScript（ビルドツールなし） |
| データストア | Google スプレッドシート（既存シート） |
| クライアント↔サーバー通信 | `google.script.run`（GAS 固有の RPC） |
| Excel 出力 | SheetJS (CDN: `xlsx@0.18.5`) |
| 認証 | GAS のデプロイ権限に従う（Google アカウント） |

## スプレッドシート構造

- **スプレッドシート ID**: `1GEfGO4_t4l5bmftBA2pXncZ1kXdMYtZBhdsvIjz4L-o`
- **シート名**: `Sheet1`
- **データ開始行**: 2行目（1行目はヘッダー）

| 列インデックス | 列名 | 型 | 例 |
|---|---|---|---|
| 0 (A) | 日付 | 文字列 "YYYY年M月D日" | `2026年5月12日` |
| 1 (B) | 品名 | 文字列 | `人参` |
| 2 (C) | 数量 | 数値 | `3` |
| 3 (D) | 単価 | 数値 | `200` |
| 4 (E) | 金額 | 数値（数量×単価） | `600` |
| 5 (F) | 事業 | 文字列 | `農場` |
| 6 (G) | 納品先 | 文字列 | `なのはな食堂` |

## GAS サーバー関数（API）

クライアントから `google.script.run` 経由で呼び出す。
クライアント側では Promise ラッパー `gasCall(fnName, arg)` を使用。

### `getMetadata()` → `{ products, businesses, destinations }`

起動時に1回だけ呼び出す軽量関数。シート全データをスキャンして返す。

```js
{
  products: {
    "人参": [
      { date: "2026年4月10日", unitPrice: 200 },
      { date: "2026年5月1日",  unitPrice: 250 }
    ],
    // ... 全品名
  },
  businesses:   ["農場", ...],      // 重複排除・ソート済み
  destinations: ["なのはな食堂", ...] // 重複排除・ソート済み
}
```

### `getSheetData(month)` → `Array<Row>`

月指定（`"2026-05"` 形式）でフィルタしたデータを返す。
一覧・請求書ビューで使用。キャッシュあり（月が同じなら再取得しない）。

各行は配列: `[date, product, qty, unitPrice, amount, business, destination, rowNum]`
- index 7 (`rowNum`) はスプレッドシートの実際の行番号（2始まり）。編集・削除に使用。

### `appendRows(rows)` → `Number`

`rows`（2次元配列）をシート末尾に追記する。
現在のアプリでは1品目ずつ `[[date, product, qty, price, amount, biz, dest]]` を渡す。

### `updateRow({ rowNum, rowData })` → `true`

指定行番号の行を `rowData`（7要素配列）で上書きする。

### `deleteRow(rowNum)` → `true`

指定行番号の行を削除する。削除後は後続行の行番号がシフトするため、
クライアントのキャッシュを無効化（`cachedMonth = null`）して再取得する。

## クライアント側の状態管理

```js
var productHistory = {};  // { "品名": [{date, unitPrice}] } — getMetadata() で構築
var businesses     = [];  // 事業一覧
var destinations   = [];  // 納品先一覧
var cachedRows     = [];  // 現在表示中の月のデータキャッシュ
var cachedMonth    = null; // キャッシュ対象の月 ("2026-05" 形式)
```

永続化（localStorage）:
- `fda_lastBusiness` — 最後に選択した事業
- `fda_lastDestination` — 最後に選択した納品先

## 画面構成（3ビュー）

### 1. 入力ビュー（デフォルト）

1枚のカードに全フィールドをまとめた構成。

```
┌─ 入力カード ──────────────────────────────────┐
│  日付: [date input]                           │
│  事業: [select]    納品先: [select]           │
│  品名: [text + オートコンプリート候補]         │
│  数量: [number]  単価: [number]  金額: [自動] │
│  (保存成功/失敗メッセージ)                    │
└──────────────────────────────────────────────┘
固定フッター: [シートに保存]
```

**動作**:
- 「シートに保存」ボタン押下で `appendRows` を1品目分呼び出し、即座にシートへ書き込む
- 保存後: 品名・数量・単価・金額をクリア、**日付・事業・納品先は保持**（連続入力のため）
- 事業・納品先の選択値は localStorage に保存

**品名オートコンプリート**:
- `productHistory` を前方一致でフィルタ
- 候補リストに品名・最新単価・最終入力日を表示
- 候補選択で単価を自動補完、フォーカスを数量欄に移動
- iOS キーボード表示時は `visualViewport` を参照して候補リストを入力欄の上に表示

### 2. 一覧ビュー

- 月選択・納品先フィルタ・事業フィルタ
- 選択月のデータをテーブル表示
- 各行に「編集」ボタン → 編集モーダル（ボトムシート）
- 件数・合計金額を表示

**編集モーダル（ボトムシート）**:
- 日付・品名・数量・単価・事業・納品先を編集
- 「保存」→ `updateRow` 呼び出し
- 「削除」→ confirm ダイアログ → `deleteRow` 呼び出し
- 保存・削除後に一覧を自動リロード

### 3. 請求書ビュー

タブ構成:

**請求書タブ**:
- 月・納品先を選択して「表示」
- 御請求書フォーマットでプレビュー
- 「印刷/PDF」→ `window.print()`
- 「Excel出力」→ SheetJS で xlsx ダウンロード（請求書シート＋月次サマリーシートの2シート構成）

**月次サマリータブ**:
- 同じ月の全データを事業×納品先でグループ集計
- 金額合計と総計を表示

## 日付フォーマット

- **スプレッドシート保存形式**: `"2026年5月12日"`（年を含む文字列）
- **date input の値**: `"2026-05-12"`（ISO 形式）
- **変換関数**:
  - `formatDateJP("2026-05-12")` → `"2026年5月12日"`
  - `dateJPToISO("2026年5月12日")` → `"2026-05-12"`
- スプレッドシートの Date オブジェクトは GAS 側 `normalizeDate_()` で文字列に正規化して返す

## デプロイ方法

1. Google スプレッドシートを開く → 拡張機能 → Apps Script
2. `コード.gs` / `index.html` / `style.html` / `script.html` の4ファイルを作成・貼り付け
3. 「デプロイ」→「新しいデプロイ」→ 種類: ウェブアプリ
4. アクセスできるユーザー: 自分のみ（または組織内）
5. 発行された URL を iPhone のホーム画面に追加

## 連携・統合の考慮点

- **データの入口**: スプレッドシート（Sheet1）の A〜G 列。外部システムがこのシートに行を追記すれば、アプリ側で即座に参照可能
- **データの出口**: スプレッドシートを直接参照する、または `getSheetData(month)` で月次データを取得
- **認証**: GAS Web App の認証に依存。外部システムから API 的にアクセスする場合は GAS の `doPost()` を追加して API キーや OAuth で保護する設計が必要
- **リアルタイム性**: アプリ起動時にメタデータをキャッシュ、月切り替え時にデータ取得。リアルタイム同期は行っていない
- **Excel 出力**: 請求書（納品先別）＋月次サマリー（事業×納品先）の2シート xlsx
